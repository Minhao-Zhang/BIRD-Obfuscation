# 面向文本到 SQL 的抗污染混淆基准:方法与设计综述

## 摘要

大规模语言模型在公开的文本到 SQL(text-to-SQL)基准上表现优异,但由于此类基准很可能已进入模型的训练语料,其高分可能部分来源于对基准内容的记忆,而非对当前数据库模式(schema)的真实理解。这一现象通常称为**数据污染(data contamination)**,它会系统性地高估模型的真实能力。本项目在广泛使用的 BIRD 基准之上,构建了一套**语义保持的混淆版评测数据集**:在不改变题目难度与正确答案的前提下,系统性地改写模型最可能借以"辨认"记忆内容的表层线索,并通过一套双向执行等价性检验保证数据集的正确性。数据集沿三个相互正交、可独立开关的维度进行混淆,并配套一项消融实验,用以分别度量各维度对模型执行准确率的独立影响。

## 1. 研究背景与动机

文本到 SQL 任务要求模型在给定数据库模式与一个自然语言问题的条件下,生成一条可正确执行、返回目标结果的 SQL 查询。评价指标通常采用**执行准确率(execution accuracy)**,即模型生成的查询与标准答案(gold SQL)在真实数据库上执行所得结果集是否一致。

BIRD 是该任务中被广泛采用的基准之一,包含数十个真实数据库与逾万条带标注 SQL 的问题。然而,作为一个长期公开的资源,其模式标识符、问题表述乃至标准答案很可能已被前沿模型在预训练阶段接触。据此,模型在原始 BIRD 上的成绩难以区分两种来源:一是对当前所给模式的真实推理能力,二是对已见基准内容的记忆性复现。若不加以控制,后者将导致对模型能力的高估,并削弱基准的判别效力。

## 2. 总体方法

本项目的核心思路是:**在保持题目语义与可解性不变的前提下,对基准施加受控的表层扰动,并通过对比模型在扰动前后的表现,估计其对记忆性线索的依赖程度。** 扰动越是削弱表现,越说明原始成绩中记忆成分的占比更高。

方法遵循两条基本约束:

- **语义与可解性保持**:每一处扰动均不改变问题的含义与其标准答案;数据库中的真实数据不作任何修改。
- **可机器验证**:每一处扰动后,均以自动化手段确认题目仍可被正确回答、且标准答案的执行结果保持不变。

在原始的 80 个数据库中,本项目保留了 **69 个**(剔除题量过少者),经端到端验证后得到 **10,164 条**有效问题(训练集 8,134 条,测试集 2,030 条),覆盖全部 69 个数据库。

## 3. 混淆维度与数据集构成

混淆沿**三个相互独立的维度**展开,各维度针对模型的一类不同记忆机制:

1. **标识符改名(identifier renaming)**——将数据库中的表名与字段名翻译为另一种语言(本项目采用英、法、德、西班牙语与汉语拼音五种)。此举削弱模型对熟悉标识符的直接辨认,迫使其依赖当前模式进行模式链接(schema linking)。
2. **诱饵注入(decoy injection)**——向数据库中引入貌似合理、实则错误的干扰性表与字段,以诱导模型偏离真实目标(详见 3.1 节)。
3. **问题改写(question paraphrasing)**——在保持语义与标准答案不变的前提下重述每一道问题,破坏模型对特定问句形式与"问句—SQL"模板的记忆性依赖。

为分别量化各维度的作用,本项目构建了**四套内容完全一致、仅混淆程度不同**的数据库实例:原始版、仅改名版、仅诱饵版,以及改名与诱饵并施版。四套实例中的真实数据逐字节一致,差异仅在于标识符命名与诱饵的有无。

### 3.1 诱饵的设计:从空置干扰项到腐化陷阱

诱饵的设计经历了一次关键调整,其动机与所服务的评测范式直接相关。

**初始方案**为引入**空置**的干扰表与字段。在仅向模型提供剥离后的模式描述(即仅含表名、字段名与数据类型)的评测设定下,空置诱饵不占存储且难以从描述中识别,可充当有效干扰。

**问题在于**,本数据集所服务的下游对象是一类**交互式"执行—观察"智能体(execute-and-observe agent)**:此类系统会主动执行探索性查询并据其结果调整策略。在该范式下,空置诱饵极易暴露——一次简单的计数查询即可揭示其为空,从而丧失干扰效力。

**据此,本项目将诱饵重构为"腐化陷阱"**:诱饵不再为空,而是承载**貌似合理但实为错误**的数据,使其由被动干扰项转变为主动陷阱——一旦模型误用该诱饵作答,即会得到错误结果。具体实现为:将某一真实字段整列复制,赋以合乎语义的近义名称,继而对副本施加受控腐化。腐化算子依字段类型而定,例如:对充当连接键的字段仅施加**整列置换**(每一取值仍为数据库中真实存在的键,故引用完整性得以保持,表面难以察觉,但行间对应关系已被打乱);对数值字段施加稀疏的相对扰动;对文本字段施加域内类别替换;对时间字段施加日期偏移。

诱饵注入遵循一条**严格的可加性(strictly additive)不变量:真实数据在任何环节均不被修改,腐化仅作用于新增的副本**。由此带来两重保障:其一,原始正确答案在任一实例中均可查得;其二,由于标准答案仅引用真实字段而从不引用诱饵,题目的可解性天然成立。

在最终数据集中,腐化诱饵以两种粒度呈现:新增的**"伪装"字段**(约 1,486 例)与整表复制并腐化的**克隆表**(162 张,覆盖 66 个数据库)。为控制存储与计算开销,仅对规模低于设定阈值的表施加相应操作。

## 4. 完整性验证

由于数据集经历了多重改写,必须以形式化手段确保**每一道题仍为原题、其标准答案仍然正确**。本项目采用**双向执行等价性检验**:将同一条标准答案分别在**原始数据库**与**改写后数据库**上执行,要求两者返回的结果集完全一致。该检验对全部题目与全部数据库变体均予执行;仅当两侧结果一致时,方判定该次改写未破坏题目。

此外,本项目通过对每一列计算与顺序无关的指纹,逐列核验四套实例中真实数据的逐字节一致性,以证实"可加性不变量"确实得到遵守。

需要说明的是,存在一类**良性且可预期**的差异:部分标准答案本身未完全约束结果顺序(例如"取前若干条"而未指定排序),此类查询在数据经重新组织后可能返回**不同但同样正确**的结果子集。本项目将此类问题单独标注并在严格计分时予以排除,以免将正常的顺序不确定性误判为错误。

## 5. 实验设计

基于上述四套数据库,评测以**消融(ablation)**方式展开:令同一模型在不同混淆程度的实例上作答同一批题目,比较其执行准确率。各对照组分别隔离一个维度的作用:

- **仅改名 vs. 原始**:度量模型对可辨认标识符的依赖;
- **仅诱饵 vs. 原始**:度量模型受错误干扰项误导的程度;
- **仅改写 vs. 原始**:度量模型对记忆性问句形式的依赖;
- **三维并施 vs. 原始**:度量最强混淆条件下的整体表现。

所有对照均在**无提示、单轮(不提供反馈、不允许重试)**的设定下进行。评分采用最严格的判据:实际执行模型生成的查询,并与标准答案的结果集作精确比对。鉴于各效应量级较小,统计推断采用**配对检验(McNemar)结合自助法(bootstrap)置信区间**,而非仅依赖点估计。需强调,该设计为"逐维度加全维度",并非完整的三因素析因设计,故不能将全维度效应简单分解为各维度效应之和。

本节的定量结果将在完成全量评测后补充。为确保结论稳健,评测将在服务器上以更强的模型对全部对照重新运行一遍;在此之前,本文不报告任何初步数值。

## 6. 设计理念与讨论

综合上述设计,可归纳出若干贯穿始终的原则:

1. **对结论保持审慎与诚实**。已有研究提示,更能暴露记忆性依赖的往往是**问句形式**而非标识符名称。据此,本数据集**最具持久价值的定位,是一套经严格验证的多语言文本到 SQL 资源与鲁棒性测试床**;污染度量则是一项需以充分实验支撑、且不宜过度诠释的次要目标。
2. **威胁模型决定设计形态**。由于服务对象为可主动查询数据的交互式智能体,诱饵必须"可被查验且查验结果为误",而不能为空置摆设。设计应贴合对手的实际探测方式。
3. **绝不破坏原件**。"可加性不变量"是同时保障"原始答案始终可达"与"题目始终可解"的根本前提。
4. **可复现性**。所有腐化均由确定性规则(基于行键与固定盐值的哈希)生成,不依赖真随机;在任何环境中重建,均可得到完全一致的数据集。
5. **面向再利用的交付**。成果按体量拆分交付:规模较大的数据库以标准数据库转储形式发布于公开托管平台;规模较小的题目、标准答案、命名映射与诱饵清单则随源码一并进行版本管理,以支持第三方下载、还原与完整复现。

## 7. 交付物与可复现性

本项目的最终交付物包括:**一套四种混淆程度、内容一致且可执行的关系数据库实例**,以及配套的**逾万条经双向验证的问题及其标准答案、完整的标识符改名映射,与逐一记载每个诱饵"真身来源及其腐化方式"的清单**。凭借这些材料,研究者可在自有环境中复现本文所述评测,或将其接入自研智能体以开展鲁棒性测试。

## 8. 局限与展望

本工作存在若干需明确披露的局限(完整清单见
[reference/limitations.md](reference/limitations.md)),其中较重要者:

- **改名维度存在"控制组"稀释**:14/69 个数据库被指派为英语,其改名映射为恒等映射(即不改名),约占全部题目的 23%。这是**有意的噪声基线控制**,但意味着**改名效应必须按语言分别报告,不能汇总成单一数值**(汇总值会被这 23% 的零效应结构性稀释);且这些库仍接受完整的诱饵与改写处理,并非"完全未混淆"。
- **结果比较偏宽松**:默认比较器会做类型归一(`1`、`"1"`、`True` 视为相等),这对各臂对称、因而在**差值(delta)中相互抵消**,但会**高估绝对准确率**。评测现已**同时报告一个严格比较下的 EX**(不跨类型归一、大小写敏感)作为保守下界;报告绝对准确率时应引用严格值。
- **验证纳入受执行预算约束**:金标准需在 300 秒超时与 20 万行上限内执行通过;行数上限是确定性的,超时在边界附近非确定,但仅影响极少数退化查询。
- **表层性**:混淆主要作用于模式与问句的表层,未处理对具体字面量或高层 SQL 模板的记忆;诱饵的实际"诱陷率"需由下游交互式评测框架进一步度量。

大规模全量评测正以更强的模型进行中,本文暂未报告定量结果;完成后将补充,并按上述口径(按语言分列、严格/宽松 EX 并列)呈现。
